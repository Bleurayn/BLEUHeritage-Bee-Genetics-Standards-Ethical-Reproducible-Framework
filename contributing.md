# Contributing to BLEUHeritage Standards

We welcome contributions to improve the BLEUHeritage Bee Genetics Standards!

## How to Contribute

### 1. Reporting Issues
- Use GitHub Issues to report bugs or suggest improvements
- Include specific examples and context
- For security issues, email security@bleuheritage.org

### 2. Suggesting Changes
- Open an issue first to discuss proposed changes
- Provide rationale and use cases
- Reference relevant standards or regulations

### 3. Submitting Pull Requests
1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Update documentation as needed
5. Submit pull request with description

## Contribution Guidelines

### For Documentation
- Follow Markdown best practices
- Include examples where helpful
- Update related documentation when making changes
- Maintain consistent terminology
### For Schemas and Protocols
- Maintain backward compatibility when possible
- Include validation rules for new fields
- Update examples to reflect changes
- Consider privacy and security implications

### For Code Examples
- Include comments explaining logic
- Handle errors gracefully
- Follow Python/industry best practices
- Include test cases where appropriate

## Review Process

1. **Initial Review**: Within 7 days of submission
2. **Feedback**: We'll provide constructive feedback
3. **Revision**: Opportunity to address feedback
4. **Decision**: Accept, request changes, or reject

## Areas of Focus

We particularly welcome contributions in:

### High Priority
- Additional validation rules
- Integration examples with other standards
- Case studies and implementation examples
- Translation of documentation

  ### Medium Priority
- Extended metadata fields for specific use cases
- Additional protocol variations
- Performance improvements to examples
- Accessibility improvements

### Low Priority
- Formatting and style changes
- Minor typo corrections
- Cosmetic improvements

## Code of Conduct

All contributors must adhere to our Code of Conduct:

- Be respectful and inclusive
- Focus on constructive feedback
- Respect different perspectives and experiences
- Gracefully accept constructive criticism
- Show empathy towards other community members
